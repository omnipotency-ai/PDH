## Critical Issues

| #   | File                | Line/Function                   | Description                                                                                                                                                                                                                                                                                                                    | Suggested Fix                                                                                                                                                        |
| --- | ------------------- | ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| C1  | `sync.ts`           | `toSyncedLogs`, line 73         | **Double `as unknown as SyncedLog[]` cast** bypasses the TypeScript type system entirely. The comment admits it "can't be verified statically." Any shape mismatch between Convex's flat union return and the discriminated `SyncedLog` type would be silently swallowed at runtime.                                           | Define a validated mapping function that explicitly narrows each `type` to its `data` shape, or use a Zod/validator schema to narrow the union before casting.       |
| C2  | `sync.ts`           | `sanitizeLogData`, line 50–51   | **`as unknown as ConvexLogData`** — unsafe chain cast at a sanitization boundary. The sanitized object bypasses type checking before being sent to Convex.                                                                                                                                                                     | Accept `LogPayloadData` and return the same type; rely on server validation. If types genuinely differ, document the exact structural delta and use a typed mapping. |
| C3  | `sync.ts`           | `asConvexId`, line 28–31        | **`return id as Id<T>`** — no non-empty string guard before the cast. `asConvexId("")` would produce a malformed ID with no error.                                                                                                                                                                                             | Add a non-empty string guard before the cast to prevent malformed IDs.                                                                                               |
| C4  | `habitTemplates.ts` | `validateHabitConfig`, line 590 | **`dailyCap === 0` is silently discarded** — `typeof habit.dailyCap === "number" && habit.dailyCap > 0` drops zero-cap habits. A cap of 0 (zero-tolerance, e.g., "no alcohol") is a valid medical constraint that gets deleted. `isCapHabit` checks `dailyCap !== undefined`, so a zero-cap habit is treated as having no cap. | Change to `habit.dailyCap >= 0` to preserve zero-cap constraints.                                                                                                    |
