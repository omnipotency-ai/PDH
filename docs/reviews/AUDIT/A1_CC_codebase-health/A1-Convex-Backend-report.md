## Auth Coverage Summary

All public queries and mutations reviewed. **Every authenticated endpoint has auth enforcement** (either inline `getUserIdentity` or `requireAuth`). Two intentionally unauthenticated public endpoints:

- `waitlist.join` — unauthenticated by design (public signup)
- `waitlist.unsubscribe` — unauthenticated by design

One concern: **`matchUnresolvedItems` action has no auth check** (the action accepts `apiKey` and `logId` but never calls `requireAuth`). Currently safe because the handler is a no-op stub, but this must be fixed before the handler is activated. See M16.

All inline `getUserIdentity` patterns are functional but inconsistent with the `requireAuth` helper. See L1/L2.
