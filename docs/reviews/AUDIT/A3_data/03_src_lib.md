## Surprising Findings

- **`src/lib/sync.ts` imports from React and Convex/React hooks.** This is a lib file that contains ~30 React hooks (`useQuery`, `useMutation`, `useAction`, etc.). It straddles the line between "library" and "hooks layer." It should arguably live in `src/hooks/sync/` or be split so that pure types (`SyncedLog`, `FoodLibraryEntry`) stay in lib and hooks move to `src/hooks/`.

- **`src/lib/habitIcons.tsx` is a `.tsx` file in lib.** It imports from `lucide-react` (a UI component library). While it only returns data (Icon component references + class strings), not JSX, it couples the lib layer to a UI icon library. This is a mild violation of the "lib should not import UI layer" principle.

- **`src/lib/habitCoaching.ts` has 8 exported functions, none of which are imported outside of `src/lib/`.** The file appears to be entirely dead or its consumers were removed during refactoring. The `AiSuggestionsCard.tsx` component imports `generateSettingsSuggestions` and `heuristicSuggestions`, but the coaching and snippet functions (`generateCoachingSnippet`, `getHeuristicCoachingMessage`, `generateHabitSnippet`, `heuristicHabitSnippet`) have zero external consumers.

- **`src/lib/featureFlags.ts` is entirely unused.** The sole flag (`transitMapV2: true`) is never read. The feature it gates appears to have shipped.

- **`src/lib/healthProfile.ts` is test-only.** Both exported functions are only consumed by their test file. No runtime code uses them. Either the consuming code was removed or this is pre-built for a future import/migration feature.

- **`src/lib/streaks.ts` exports dead gamification types.** `GamificationState` and `DEFAULT_GAMIFICATION` are defined but never imported. Only `SleepGoal` and `DEFAULT_SLEEP_GOAL` have consumers. The gamification system may have been descoped.

- **`src/lib/foodStatusThresholds.ts` has 11 exports but only 3 are imported externally** (`Zone`, `computeBristolAverage`, and `clampZone`). The remaining 8 constants (`MIN_RESOLVED_TRIALS`, `RISKY_BAD_COUNT`, `WATCH_BAD_COUNT`, `BRISTOL_HARD_UPPER`, `BRISTOL_LOOSE_LOWER`, `ZONE_MIN`, `ZONE_MAX`) and `classifyConsistency` are either dead or were migrated to `shared/foodEvidence`.

- **`src/lib/analysis.test.ts` is a co-located test file** (not in `__tests__/`). This is inconsistent with the other 12 test files which live in `src/lib/__tests__/`. Move it for consistency.

- **`src/lib/reproductiveHealth.ts` has 4 dead exports** (`isBleedingStatusActive`, `bleedingStatusBadgeClass`, `getDateKeyFromTimestamp`, `getTodayDateKey`). The latter two are trivial wrappers around `formatLocalDateKey` that add no value.

- **`src/lib/aiAnalysis.ts` at 1,954 lines is by far the largest file** in the entire `src/lib/` directory and exceeds the 300-line threshold by 6.5x. It contains the entire Dr. Poo AI system: a 1,200-line system prompt builder, log context assembly, insight parsing, weekly summary generation, and educational insight deduplication. This is the highest-priority decomposition target.
