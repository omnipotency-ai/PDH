### Circular Dependencies

**No true circular dependencies found.**

The import graph flows cleanly in one direction:

```
store.ts (leaf -- imports only from lib/ and types/)
    ^
    |
contexts/ (import from hooks/, store, lib/)
    ^         \
    |          v
hooks/ (import from contexts/, store, lib/)
```

**Bidirectional (but acyclic) relationship:** `hooks/` and `contexts/` import from each other, but specific file-to-file chains never form a cycle:

- `useProfile` -> `ProfileContext` -> convex (no cycle)
- `SyncedLogsContext` -> `useProfile` -> `ProfileContext` (no cycle)
- `ApiKeyContext` -> `useApiKey` (no cycle)
- `useAiInsights` -> `ApiKeyContext` -> `useApiKey` (no cycle)

### Boundary Violations

1. **convex/ importing from src/ (server importing from client)**
   - `convex/extractInsightData.ts` imports `StructuredFoodAssessment` from `../src/types/domain` -- type-only import with no runtime impact, but creates fragile coupling. The type should be in `shared/` or duplicated as a convex validator.

2. **hooks importing from components (inverted dependency)**
   - `src/hooks/useUnresolvedFoodQueue.ts` imports `getFoodItemResolutionStatus` from `@/components/track/today-log/helpers` -- pure utility that should live in `src/lib/`
   - `src/hooks/useFoodParsing.ts` imports `ParsedItem` type from `@/components/track/panels` -- type that should be in `src/types/domain`

3. **components importing directly from convex/ source (bypassing hooks layer)**
   - `src/components/track/FoodMatchingModal.tsx` imports `api` from `../../../convex/_generated/api` and calls `resolveItem`, `submitFoodRequest`, `searchFoods` mutations directly
   - `src/components/settings/AiSuggestionsCard.tsx` imports `api` from `../../../convex/_generated/api` and calls `useAction` directly
   - `src/components/landing/WaitlistForm.tsx` and `PricingSection.tsx` also import from `convex/` directly (acceptable for landing forms but inconsistent)

4. **lib importing from UI layer**
   - `src/lib/habitIcons.tsx` is a `.tsx` file that imports from `lucide-react` (a UI component library) -- couples the lib layer to a UI icon library
   - `src/lib/sync.ts` contains ~30 React hooks (`useQuery`, `useMutation`, etc.) -- straddles lib and hooks layers

5. **src/types/ importing from src/lib/ (types depending on implementation)**
   - `src/types/domain.ts` imports from 4 `src/lib/` modules (`aiModels`, `habitTemplates`, `streaks`, `units`) -- prevents domain.ts from being shared with convex/
