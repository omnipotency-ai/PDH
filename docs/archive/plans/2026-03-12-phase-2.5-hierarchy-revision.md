# Phase 2.5: Food Hierarchy Revision — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the flat `TransitLine` type with a two-level `FoodGroup` + `FoodLine` hierarchy across the entire food system, delete legacy bridge code, and fix all consumers.

**Architecture:** The food registry gains two new required fields (`group: FoodGroup`, `line: FoodLine`) replacing the single `line: TransitLine`. Zone 1A liquid entries (water, tea, juice, supplement) are removed from the registry entirely — they are pre-map basics learned in hospital. Broth and soup stay but get real group/line assignments. All legacy `LineCategory` usage is replaced with `FoodGroup` from the registry. Legacy transit map files are deleted.

**Tech Stack:** TypeScript, React, Vitest, Biome formatter

## Key design decisions

- **No optional fields**: `group` and `line` are REQUIRED on every registry entry
- **6 entries removed**: water, apple juice, pear juice, tea, nutritional supplement, sugary drink
- **Broth and soup stay**: clear broth → protein/meat_fish; smooth soup → carbs/vegetables
- **Milk moves**: from hub to protein/eggs_dairy (not a liquid)
- **Kefir added**: protein/eggs_dairy, Zone 3
- **Fried egg**: Zone 3 (not Zone 2 — fried = added fat, user confirmed Zone 3)
- **Garlic**: seasoning/herbs_spices (it's a herb)
- **Onion**: carbs/vegetables (it's a vegetable)
- **Dairy split**: yogurt/cottage cheese/milk/kefir → Protein; butter/cream/cheese → Fats
- **Avocado**: fats/nuts_seeds
- **No backwards compatibility** with legacy types
- **Authoritative line orders**: Follow `food-registry-restructured.md` tables exactly

```typescript
export function getFoodsByLine(
  line: FoodLine,
): ReadonlyArray<FoodRegistryEntry> {
  return FOOD_REGISTRY.filter((e) => e.line === line).sort(
    (a, b) => a.lineOrder - b.lineOrder,

export function getLinesByGroup(group: FoodGroup): ReadonlyArray<FoodLine> {
  return GROUP_LINES[group];
```

```typescript
export function buildRegistryVocabularyPrompt(): string {
  const lines: string[] = [];
  lines.push("canonical | zone | group | line | examples");
  lines.push("--- | --- | --- | --- | ---");

  for (const entry of FOOD_REGISTRY) {
    const exampleList =
      entry.examples.length > 0 ? entry.examples.slice(0, 5).join(", ") : "";
    lines.push(
      `${entry.canonical} | ${entry.zone} | ${entry.group} | ${entry.line} | ${exampleList}`,
    );
  }

  return lines.join("\n");
}
```
