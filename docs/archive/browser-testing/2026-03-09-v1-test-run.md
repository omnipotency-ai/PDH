## Phase 2: Track Page — Logging

Can the user log one of each entry type and see it appear in the today log?

- [x] **Food** — PASS with issues: AI parses and appears in today log. Time field shows first but is auto-timestamped to now; if you click time before entering food it errors saying enter food first (unnecessary friction). No confirmation modal anymore (good — removed friction). BUT parsing is synchronous, blocks UI for 3-5 seconds. Should be async (fire-and-forget).
- [x] **Bowel movement** — PASS: most fully realised panel. Bristol type selection works, save works, appears in log. Issue: mystery time display at bottom — looks like time since last BM but unlabelled. Looks clickable (like you could edit BM time) but isn't. Should be able to edit BM timestamp.
- [x] **Fluid** — PASS with issues: presets work, appears in log. BUT the preset-with-amount model is wrong — if you want preset+amount, use a quick capture card. Fluid section should be for precise logging: just drink name + ml input (like it was before). Current design forces choosing preset amounts (250ml) when you actually drank a different amount.
- [x] **Habit** — PASS: quick capture cards work. Count increments register. Sleep, bowel habits, water, coffee (reduces remaining count), tea, walking all work. Destructive habits (cigarettes, sweets, alcohol) countdown correctly. Add habit works (known from prior use).
- [x] **Weight** — PASS with issues: click weigh-in card → type weight → enter → saves → updates log → persists on card day-to-day. Weight Settings drawer: has starting/current/target weight. Target weight defaults to selected state with orange system highlight (ugly). "Auto-saves on change, leave blank to clear" — just put a "log weight" button instead. Stats show "since last weigh-in" and "since surgery" but no dates. Weight trend graph: only shows 23 Feb and 8 Mar on x-axis, shows target line, but looks incomplete — no progress-toward-goal visualisation.
- [ ] **Activity** — PASS (functional) with design issues: no dedicated activity panel. Activities added via "Add habit" button, appear as quick capture cards. Long-press opens detail sheet showing: progress bar, last 7 days, daily target, "quick duration" setting (now redundant since input replaced click-to-increment). Weekly target field exists but does nothing — visual-only with no measurement. Sheet only dismissable via X button — should support Escape key, click outside, swipe down on mobile.

**Notes:** Food parsing blocking the UI is the biggest UX issue. Weight drawer needs a redesign. Activity needs its own section or at least the detail sheet cleaned up.

## Phase 3: Today Log (right column)

Does the log display entries correctly and allow inline editing?

- [x] **Entries grouped correctly** — PASS
- [x] **Tap to expand** — PASS
- [x] **Edit and save** — PASS
- [x] **Delete an entry** — PASS
- [x] **Date picker** — PASS: changes date, shows correct day's entries

**Notes:**

- Completed boolean habit logs use a square checkbox with group-colour background + white tick. Should be a round emerald green circle with white tick (matching quick capture card style) for uniformity and a splash of colour in the log.
- Weigh-in entry should be pink in the log (icon + weight value) to match the pink quick capture card.
- Time picker: inline keyboard/mouse input is clunky. Consider opening a clock-face picker (hours + minutes) instead, or at minimum improve the inline input feel.
- Can't add entries for past dates. Date picker navigates the log view but quick capture always logs to today. Ideally quick capture would be linked to the selected log date — but risk of accidentally logging to wrong day. No clear solution yet; works as-is for v1.0.
- Sleep split across midnight: logging 7h sleep at 5am attributes all 7h to today, but should split 5h to today and 2h to yesterday (bed at 10pm, wake at 5am). This is a data correctness issue for pass 2.

## Phase 4: Patterns Page

Do the analytics panels show real data from logged entries?

- [ ] **Hero strip** — FAIL: renders but data is not useful. See notes.
- [ ] **Food safety grid** — FAIL: safe today / next food to unlock / menu card — content is not useful. See notes.
- [x] **Database tab** — PASS with major issues: table loads, columns visible, data rows present. But status logic, zones, and deduplication are broken. See notes.

- [ ] **Filter/search** — PARTIAL: filter sheet has open/close/reopen flicker bug. Filter applies but missing statuses (e.g. "safe-loose" not in options). Multi-select, free-text search, and clear filter work.

**Notes:**

**Hero strip:**

- Bristol trend: squiggly line with no x-axis, no y-axis, no hover tooltips. Can't tell what any point means. Charts are not useful.
- BM count: shows "0 today" — redundant with Track page. Should show 7-day rolling average instead. Also displays zero as "naught" on the sparkline. Today shouldn't be counted since it's still in progress.
- Transit score: shows "250, stage 2, 5 of 92 cleared" with a stacked colour bar. No explanation of what 250 means, what stage 2 is, what the 5 cleared items are, what the colours represent.
- Next trial window: "opens in 7h 8m" — self-evident, could be improved but works.

**Food safety grid (3-card layout: Safe Today / Next Food to Unlock / Menu):**

- Safe Today: just lists "white toast, bananas, scrambled eggs" with generic advice "start with a small portion and keep meals frequent" — not personalised or useful.
- Next Food to Unlock: "182 foods still testing, add one test food every 2 days" — generic. Lists onion as a next food, but onion is a derivative ingredient from guacamole — should be guacamole, not onion.
- Menu card: this is the only way to access the Menu page. Menu page content is broken — see below.
- Habit-digestion correlations: expanded by default (should be collapsed). Correlation logic is too simplistic — e.g. says more sleep = worse BM quality, but that's because bad BM nights cause next-day sleep catch-up, not causal. Walking/cigarette correlations inconclusive.

**Menu page (accessed from food safety grid):**

- Shows 10 safe, 182 building, portion guidance, meal slots (7-9, 10-11, 12-2, 3-4, 6-8, 8-9) — slots are wrong for user, probably hardcoded or from unconfigured settings.
- Next foods to unlock: zone assignments wrong (boiled chicken = zone 2, should be zone 1; pumpkin = zone 3, should be zone 1-2; applesauce = zone 3, should be zone 1).
- Generic advice repeated for every food: "try small portion, 2-3 tbsp, batch cook, keep stable 6+ hours" — identical for all, not useful.
- Full 182-food list is unreadable. UI is broken.

**Database:**

- Duplicate food entries present (e.g. "Cottage Cheese" plain, "Grams Of Cottage Cheese", "G Cottage Cheese" — all separate entries).
- Status logic wrong: cheese has 2x Bristol 4, 1x Bristol 6, 1x Bristol 7 → "building" (should be "watch"). Honey has a few bad results → "avoid" (should be "watch"). Salt has 3x Bristol 4, 1x Bristol 5 → "building" (should be "safe"). Foods with 5-9 trials still "building" when they have enough data to classify.
- Categories not populated for most foods. AI column exists but empty for all.
- Trend line shows up/down but doesn't seem connected to Bayesian engine output.
- Zones assigned by food parser AI are frequently wrong.
- Clicking a food shows trial history — data is there, but status doesn't reflect it.

5. **Toast styling is minimal.** `sonner.tsx` has basic font/padding customization but no coloured backgrounds per toast type, no visible stacking config, and no prominent undo button styling.
6. **Fluid units partially wired.** `FluidSection.tsx` uses `getDisplayFluidUnit()` and `formatFluidDisplay()` for its own UI, but other surfaces (stats bar, insights, today log placeholders) may still hardcode ml.
