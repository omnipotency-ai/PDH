## Part 1: TransitMapCanvas Component

### WQ-400: TransitMapCanvas.tsx — Core SVG Rendering

Create `src/components/patterns/transit-map/TransitMapCanvas.tsx`.

This is the main component. It renders a `<svg viewBox="0 0 2000 2000">` with:

1. **Zone rings** — 4 concentric `<circle>` elements with `vector-effect="non-scaling-stroke"`, subtle colors from `ZoneRing[]`
2. **Line tracks** — 11 `<path>` elements with `vector-effect="non-scaling-stroke"`, corridor colors from `LineGeometry[]`
3. **Station circles** — `<circle>` for each station, colored by evidence status (safe=green, building=blue, watch=amber, avoid=red, untested=grey 30% opacity)
4. **Hitbox circles** — invisible `<circle r={22}>` per station for 44px touch targets (WCAG 2.5.8)
5. **Zone labels** — text along ring arcs ("Zone 1A", "Zone 1B", etc.)

Data flow:

```
useTransitMapGeometry() → paths, coordinates, rings
useTransitMapData()     → per-station evidence status
```

The component must fuse geometry (static) with evidence (reactive). Geometry is computed once and cached. Evidence updates reactively as the user logs food.

### WQ-401: useTransitMapZoom — Zoom State Machine

Create `src/hooks/useTransitMapZoom.ts`.

4-level discrete step-zoom:

| Level | Name     | Trigger                              | CSS Transform                  |
| ----- | -------- | ------------------------------------ | ------------------------------ |
| 1     | Overview | Default / pinch out / back button    | `scale(1) translate(0, 0)`     |
| 2     | Corridor | Tap empty space in a corridor region | `scale(1.6) translate(cx, cy)` |
| 3     | Line     | Tap a line track path                | `scale(2.8) translate(cx, cy)` |
| 4     | Station  | Tap a station circle                 | `scale(5.0) translate(cx, cy)` |

The hook returns:

- `currentZoom: ZoomLevel`
- `zoomTo: (level, target?) => void`
- `zoomOut: () => void` (step back one level)
- `transformStyle: CSSProperties` (the CSS transform string)

Zoom transitions use `transition: transform 300ms ease-out` on the SVG container.

### WQ-402: Station Labels — HTML Overlay

Station names are rendered as **HTML** (not SVG `<text>`) for better font rendering and wrapping. They are positioned absolutely using coordinate transforms from SVG (x, y) to screen pixels.

At each zoom level, different labels are visible:

- Overview: representative stations only (`isRepresentative: true`)
- Corridor: all representatives for that corridor
- Line: all stations on that line
- Station: station name + inline detail

Labels must track the SVG transform — when the SVG scales/pans, labels move with it.

### WQ-403: Station Focus Callout

At zoom level 4 (Station), the layout splits **side-by-side**:

- Left: zoomed SVG showing 3-4 neighboring stations
- Right: Station Inspector panel (NOT an overlay — a sibling div)

The inspector shows:

- Station name + zone badge
- Status, confidence, tendency
- Trial count + Bristol breakdown (compact)
- Digestion badges (from registry metadata)
- AI verdict (if available from `useTransitMapData`)

This reuses or adapts the existing `StationInspector.tsx` component if it exists.

**Critical rule:** The inspector NEVER overlays the map. Side-by-side split only.
